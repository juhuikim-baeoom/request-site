# 웹 프로젝트 표준 모음 — 데이터베이스 · 데이터 · 문서 관리

이 폴더는 웹 시스템을 **체계적으로 관리**하기 위한 표준 규칙 모음입니다.
외부 베스트프랙티스(SQL 스타일 가이드, 진화적 데이터베이스 설계, Diátaxis, Docs-as-code 등)를 참고하고,
운영 중인 데이터 통합 플랫폼(DataHub)의 실전 관리 기반을 **범용·비종속 형태로 재정리**했습니다.

특정 회사·스키마·경로에 종속되지 않도록 작성했으며, 예시로 제시된 목록(약어·접두사 등)은
**그대로 채택하거나 프로젝트 도메인에 맞게 조정**하면 됩니다.

## 적용 방법 (AI 코딩 에이전트에게 전달)

새 프로젝트에서 아래처럼 요청하면 됩니다. 그대로 복사해 사용하세요.

```
이 폴더(web-project-standards)의 모든 .md 문서를 읽고, 우리 프로젝트의
데이터베이스·데이터·문서 관리 표준으로 채택해줘.
- 프로젝트의 CLAUDE.md(또는 AGENTS.md)에 핵심 규칙을 요약해 반영하고,
  상세는 이 문서들을 참조하도록 링크해줘.
- 우리 스택/도메인에 맞지 않는 예시(접두사·약어 세트 등)는 조정안을 제안해줘.
- 기존 스키마/문서와 충돌하는 부분은 목록으로 먼저 보고하고 진행해줘.
```

## 문서 구성

| 파일 | 내용 | 대상 |
|------|------|------|
| `01-database-naming-rules.md` | DB 네이밍 룰 — 표준 약어, 접미사 규약, 예약어 회피, 코드/enum 네이밍 | 스키마 설계자·백엔드 |
| `02-table-column-standards.md` | 테이블·컬럼 표준 — 목적 기반 접두사, 감사 컬럼, soft-delete, PK 전략, 타입 매핑 | 스키마 설계자·백엔드 |
| `03-data-management-rules.md` | 데이터 관리 규칙 — 마이그레이션(버전·불변성·dry-run·멱등), EAV 키, 표준 사전 거버넌스, 시크릿·환경 일관성 | 백엔드·DBA |
| `04-document-management-rules.md` | 문서 관리 규칙 — Docs-as-code·SSOT, 문서-코드 동기화, 마크다운 표현(Mermaid), 언어·표기 정책 | 전체 |

## 전제 스택 (조정 가능)

| 구분 | 전제 | 다른 스택일 때 |
|------|------|----------------|
| 데이터베이스 | PostgreSQL 계열 | 예약어 목록·타입 매핑만 해당 DB로 치환 |
| 마이그레이션 | Flyway 계열(버전 파일) | Liquibase/Prisma/Alembic 등도 원칙 동일 |
| 언어 | 한국어 팀(다국어 데이터 포함) | 언어 정책 섹션만 팀 언어로 조정 |
| 문서 | 마크다운 + Mermaid | 동일 |

## 핵심 원칙 요약 (한 눈에)

- **네이밍**: `snake_case` · 소문자 · 예약어 회피 · 표준 약어 일관 사용(TP/CD/NM/UUID/AT/FLAG/SEQ).
- **테이블**: 목적 기반 접두사 · 표준 감사 컬럼 · soft-delete(물리 삭제 금지) · 안정적 PK.
- **데이터**: 마이그레이션은 불변·forward-only · 스키마 prefix 하드코딩 금지 · 운영 유사 데이터 dry-run · 멱등 시드 · 시크릿 평문 금지.
- **문서**: SSOT · 코드 변경 시 문서 동기화 · Mermaid/테이블(ASCII 박스 금지) · 단일 언어 본문(한국어는 한자 미사용).
